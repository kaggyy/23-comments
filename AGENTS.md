# UI Implementation Guidelines

このプロジェクトのUIでは、情報量を抑えたシンプルで静かな画面設計を優先する。

## 基本方針

- UIデザインを独自に拡張しない
- 指定されたデザイン、スクリーンショット、既存コンポーネントを忠実に再現する
- 指定されていないテキストやUI要素を追加しない
- 親切さを理由に説明文、ヘルプ文、注意文、件数、タグ、チップ、カードを増やさない
- 迷った場合は、追加するのではなく最小構成を選ぶ

## 情報量の制約

- 画面の主目的に関係しない情報は表示しない
- 一覧画面では、ユーザーが次の判断に必要な情報だけを表示する
- 同じ意味を、ラベル・説明文・ステータスで重複表現しない
- 既存画面より情報密度を上げない

## テキストの制約

- 新規文言を勝手に作成しない
- プレースホルダー、注釈、リード文、空状態文言を追加する場合は事前に確認する
- UI内の文章は短くし、説明過多にしない

## コンポーネントの制約

- 不要なカード化を避ける
- 不要な枠線、背景色、アイコン、バッジを追加しない
- ボタンは主要操作に絞る
- フィルター、検索、ソート、タブは明示的に依頼された場合のみ追加する

## 実装前の確認

新規UIや大きな変更では、コードを書く前に以下を提示する。

1. 表示する要素
2. 表示しない要素
3. 主要操作
4. 追加しようとしている独自判断がある場合はその内容

## 実装後のセルフレビュー

- 指定デザインにないテキストを追加していないか
- 指定デザインにないUI要素を追加していないか
- 不要なカード、チップ、アイコン、説明文がないか
- 情報を減らしても成立する箇所が残っていないか
