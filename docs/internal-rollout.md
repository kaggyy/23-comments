# 23 comments MVP 社内導入フロー

## 目的

このドキュメントは、現在のMVPを社内検証版として固定し、少人数で導入するための手順です。
管理画面はVercelにデプロイし、Chrome拡張は初期検証では手動インストールで配布します。

## 1. MVPを固定する

- 現在のUIと機能を「社内検証版」として扱います。
- 検証期間中は追加開発を止め、不具合修正だけに絞ります。
- 対象機能は以下です。
  - 表示中ページのスクリーンショット投稿
  - 矩形注釈
  - コメント管理
  - ステータス変更
  - プロジェクト作成、削除、一括削除
  - 招待リンク作成
  - メンバー表示

## 2. Supabase本番環境を用意する

1. 社内用のSupabaseプロジェクトを1つ作成します。
2. Supabase SQL editor、またはSupabase CLIでmigrationを順番に実行します。
   - `supabase/migrations/20260604000000_init.sql`
   - `supabase/migrations/20260604010000_profiles.sql`
   - `supabase/migrations/20260604020000_storage_delete.sql`
3. Storage bucket `report-assets` が作成されていることを確認します。
4. RLS policyが有効で、ログイン済みメンバーだけがデータを扱えることを確認します。
5. Supabaseの以下を控えます。
   - Project URL
   - Publishable key / anon key

## 3. 管理画面をVercelへデプロイする

1. Vercelでこのリポジトリをimportします。
2. Root directoryを `apps/web` に設定します。
3. Build commandは `npm run build --workspace @comment-tool/web` を使います。
4. Environment variablesに以下を設定します。
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
5. デプロイ後、本番URLで以下を確認します。
   - ログインできる
   - プロジェクトを作成できる
   - 招待リンクを作成できる
   - コメント一覧と詳細が表示される

## 4. Chrome拡張を検証配布する

1. 拡張機能をビルドします。

```bash
npm run build --workspace @comment-tool/extension
```

2. 検証メンバーに `apps/extension/dist` を共有します。
3. 各メンバーはChromeで以下を行います。
   - `chrome://extensions` を開く
   - デベロッパーモードを有効にする
   - 「パッケージ化されていない拡張機能を読み込む」を押す
   - `apps/extension/dist` を選択する
4. 拡張機能の設定に、管理画面と同じSupabase情報を入力します。
   - Supabase URL
   - Supabase anon key

## 5. 少人数で試験運用する

- 5〜10人程度で1〜2週間使います。
- 対象は制作、開発、PMなど、実際にWeb確認を行う人に絞ります。
- 1案件または数案件で、以下の流れを試します。
  - ページをキャプチャする
  - 矩形で指摘箇所を囲む
  - コメントを投稿する
  - 管理画面で確認する
  - ステータスを変更する
  - コメントを追記、編集、削除する
  - プロジェクト削除時に紐づくコメントが一括削除されることを確認する

## 運用ルール

- 投稿時の内容は「何を直してほしいか」が分かる文章にします。
- ステータスは以下で運用します。
  - `未対応`: 新規投稿
  - `対応中`: 担当者が確認中
  - `解決済み`: 修正完了
  - `アーカイブ`: 完了後に一覧から整理したいもの
- プロジェクト削除はコメントも一括削除されるため、検証中は管理者相当の人だけが実行します。
- MVPでは全ユーザーをowner扱いにします。権限分離は本格導入後に検討します。
- 画像やコメントに社外秘情報が含まれる可能性があるため、利用対象案件と共有範囲を事前に決めます。

## 検証チェックリスト

- [ ] 拡張機能からスクリーンショット投稿できる。
- [ ] 矩形注釈が管理画面で正しく表示される。
- [ ] 管理画面でステータス変更できる。
- [ ] 管理画面でコメント追加、編集、削除ができる。
- [ ] 招待リンクからメンバー参加できる。
- [ ] 招待モーダルでメンバー名とメールアドレスが表示される。
- [ ] プロジェクト作成、削除、一括削除が意図通り動く。
- [ ] 社外秘情報の扱いについて、社内ルールを確認した。

## 検証後の判断

検証で問題がなければ、Chrome拡張をGoogle Workspaceの社内限定配布へ移行します。
本格導入前に、以下を改善候補として確認します。

- 権限ロールの分離
- メール通知
- プロジェクト別メンバー管理
- 本番用エラーログ
- Storage容量と削除ポリシー
- Chrome拡張の自動更新配布
